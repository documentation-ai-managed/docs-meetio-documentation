## Overview

Authentication is the foundation of secure access to Meetio's platform. You use API keys and tokens to interact with communities, events, and networking features. Proper authentication ensures your data remains protected while enabling seamless integration.

## API Key Management

<Steps>
  <Step title="Generate API Key" icon="key">
    Log into your Meetio dashboard and navigate to the API settings to generate a new key.
  </Step>
  <Step title="Store Securely" icon="lock">
    Store your API key securely and never expose it in client-side code or public repositories.
  </Step>
  <Step title="Rotate Regularly" icon="refresh-cw">
    Rotate your API keys periodically and revoke compromised keys immediately.
  </Step>
</Steps>

## Authentication Methods

<Tabs>
  <Tab title="Bearer Token" icon="shield">
    Use Bearer token authentication for API requests by including the token in the Authorization header.
  </Tab>
  <Tab title="API Key" icon="key">
    Pass your API key as a query parameter or in the request header for simpler integrations.
  </Tab>
</Tabs>

## Code Examples

<CodeGroup tabs="JavaScript,Python">
  ```javascript
const axios = require('axios');

const API_KEY = 'your-api-key-here';

const headers = {
  'Authorization': `Bearer ${API_KEY}`,
  'Content-Type': 'application/json'
};

axios.get('https://api.meetio.com/v1/communities', { headers })
  .then(response => {
    console.log('Communities:', response.data);
  })
  .catch(error => {
    console.error('Authentication failed:', error.response.data);
  });
```
  ```python
import requests

API_KEY = 'your-api-key-here'

headers = {
    'Authorization': f'Bearer {API_KEY}',
    'Content-Type': 'application/json'
}

response = requests.get('https://api.meetio.com/v1/communities', headers=headers)

if response.status_code == 200:
    communities = response.json()
    print('Communities:', communities)
else:
    print('Authentication failed:', response.json())
```
</CodeGroup>

## Token Expiration

<Callout kind="info">
  Access tokens expire after 24 hours. Refresh them using your API key before expiration to maintain continuous access.
</Callout>

## Best Practices

<ExpandableGroup>
  <Expandable title="Security Guidelines" default-open="true">
    Never commit API keys to version control. Use environment variables for secure storage.
  </Expandable>
  <Expandable title="Rate Limiting" default-open="false">
    Monitor your API usage to avoid hitting rate limits. Implement exponential backoff for retries.
  </Expandable>
  <Expandable title="Error Handling" default-open="false">
    Handle authentication errors gracefully by checking response status codes and providing user feedback.
  </Expandable>
</ExpandableGroup>

## Troubleshooting

<Callout kind="danger">
  If authentication fails, verify your API key is valid and not expired. Check that you're using the correct endpoint URL.
</Callout>
